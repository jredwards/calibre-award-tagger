# Project Blueprint: Calibre Awards Tagger

## 1. Project Overview

**Objective:** Automate the enrichment of a Calibre eBook library by identifying and tagging books that have won or been shortlisted for major literary awards.
**Method:** instead of relying on stochastic AI (LLMs) to guess awards, we will build a deterministic "Canonical Awards Database" by scraping reliable sources (Wikipedia). We will then use fuzzy string matching to correlate the Calibre library against this trusted dataset.

**Target Awards:**

* Hugo, Nebula, Locus, Arthur C. Clarke (Sci-Fi/Fantasy)
* Booker, International Booker, National Book Award (Literary/General)
* Agatha Award (Mystery)
* Scotiabank Giller Prize (Canadian Fiction)

## 2. System Architecture

The system consists of two distinct pipelines that meet in the middle.

```mermaid
graph TD
    subgraph "Phase 1: Knowledge Acquisition"
    A[Wikipedia/Sources] -->|Scrapers| B[Raw Data]
    B -->|Normalization| C[Canonical Awards DB (CSV/SQLite)]
    end

    subgraph "Phase 2: Library Enrichment"
    D[Calibre Library] -->|calibredb export| E[Library JSON]
    C -->|Fuzzy Match| F[Matching Engine]
    E -->|Fuzzy Match| F
    F -->|Match Results| G[Tag Generation]
    G -->|calibredb set_metadata| D
    end

```

## 3. Tech Stack

* **Language:** Python 3.10+
* **Data Handling:** `pandas` (for table parsing), `sqlite3` (optional, CSV is sufficient for portability).
* **Web Scraping:** `requests`, `BeautifulSoup4`, `lxml` (for parsing Wikipedia tables).
* **Matching:** `rapidfuzz` or `thefuzz` (for high-performance fuzzy string matching).
* **Integration:** `subprocess` (to call `calibredb`).
* **Testing:** `pytest` for unit and integration tests.

## 4. Agent Execution Protocol (The TODO List)

This section contains the sequential tasks for the agent.

### Phase 1: Environment & Scaffolding

* [ ] **Task 1.1: Initialize Project Structure**
* Create subfolders: `scrapers/`, `data/`, `logs/`, `tests/`.
* Create `requirements.txt` with: `pandas`, `requests`, `beautifulsoup4`, `rapidfuzz`, `lxml`, `html5lib`, `pytest`.
* Create a main orchestration script `main.py` (placeholder).


* [ ] **Task 1.2: Define Data Schema**
* Define the output format for the Canonical Database.
* **Columns:** `prize_name` (str), `year` (int), `category` (str, e.g., "Best Novel"), `result` (enum: "Winner", "Shortlist", "Nominee"), `book_title` (str), `author` (str).


### Phase 2: The Scrapers (Knowledge Acquisition)

*Goal: Generate a raw CSV for each award.*

For Wikipedia, use Wikidata Query Service (SPARQL) if it is a better option in terms of reliability, accuracy, and simplicity. Otherwise scrape the pages and analyze tables.

* [ ] **Task 2.1: Scraper Logic - Generic Wikipedia Parser**
* Create `scrapers/base_wiki.py`.
* Implement a function `fetch_wiki_tables(url)` that returns a list of DataFrames.
* Implement a helper to clean citations (remove `[1]`, `[a]`) and convert years to integers.


* [ ] **Task 2.2: Scrape The "Big General" Awards**
* **Source:** Wikipedia pages for *Booker Prize*, *International Booker*, *National Book Award for Fiction*, *Scotiabank Giller Prize*.
* **Logic:** These pages usually have standard "List of winners" tables.
* **Action:** Write `scrapers/general_fiction.py` to ingest these and map columns to the schema. Handle the specific "Winner" vs "Shortlist" formatting (often indicated by background color or a specific status column).

For International Booker, we only care about the author. Nothing needed for the translator.


* [ ] **Task 2.3: Scrape The "Sci-Fi/Fantasy" Awards**
* **Source:** Wikipedia pages for *Hugo*, *Nebula*, *Arthur C. Clarke*.
* **Challenge:** These often have one table per year or massive tables split by decade.
* **Action:** Write `scrapers/scifi.py`. Ensure it captures *only* "Best Novel" categories (ignore Short Stories/Novelettes to reduce noise, unless desired).


* [ ] **Task 2.4: Scrape The "Specialized" Awards**
* **Source:** *Agatha Award* (Mystery), *Locus Award*.
* **Logic:**
* For **Locus**: Massive number of sub-categories. Filter strictly for "Sci-Fi Novel", "Fantasy Novel", "Horror Novel".
* For **Agatha**: All categories, separated. So in the tag it would be `Award: Agatha, Best First Novel [Winner]` for example.


* **Action:** Write `scrapers/specialized.py`.


* [ ] **Task 2.5: Merge & Clean**
* Create `build_db.py`.
* Load all CSVs generated by scrapers.
* **Normalization:**
* Strip subtitles (e.g., convert "Dune: A Novel" -> "Dune").
* Normalize authors ( "Ursula K. Le Guin" vs "Ursula Le Guin").

For normalizing authors, ask AI for each author what are their aliases and common names and use that. We need a special service for this with a carefully crafted prompt.


* Export `data/canonical_awards.csv`.



### Phase 3: The Matching Engine

* [ ] **Task 3.1: Calibre Extraction**
* Create `lib_interface.py`.
* Implement `get_calibre_books()` using `calibredb list --for-machine --fields id,title,authors`.
* Return a list of dictionaries.


* [ ] **Task 3.2: Fuzzy Matching Logic**
* Create `matcher.py`.
* **Step 1 (Exact Match):** Try to match simplified Title + Last Name exactly.
* **Step 2 (Fuzzy Match):** If exact match fails, use `rapidfuzz` with a high threshold (e.g., >90%).
* **Constraint:** The *Author* must match to a high degree of certainty before checking the *Title* to prevent tagging "The City & The City" by China Miéville with awards for a different book with a similar name.


* [ ] **Task 3.3: Report Generation**
* Before updating, generate `proposed_changes.json`.
* Format: `{"book_id": 123, "title": "Dune", "new_tags": ["Award: Hugo (Winner)", "Award: Nebula (Winner)"]}`.



### Phase 4: Execution & Feedback

* [ ] **Task 4.1: Dry Run Mode**
* Implement `main.py` argument `--dry-run`.
* Print a readable table of what *would* happen.
* Example output: `[MATCH] 'The Fifth Season' (ID: 45) -> Add 'Award: Hugo (Winner)'`.


* [ ] **Task 4.2: The Tagger (Write Ops)**
* Implement `apply_tags()` in `lib_interface.py`.
* Use `calibredb set_metadata {id} --tags {tag_list}`.
* Ensure tags are *appended*, not overwritten (Calibre usually appends by default, but verify flag usage).


* [ ] **Task 4.3: Final Verification**
* Run the script on the full library.
* Open Calibre GUI and filter by tag `Award: Winner` to verify results.


### Phase 5: Testing

* [ ] **Task 5.1: Unit Tests for Scrapers**
* Write tests for `base_wiki.py` helper functions (citation cleaning, year parsing).
* Write tests for each scraper's parsing logic using sample HTML fixtures.

* [ ] **Task 5.2: Unit Tests for Matching Engine**
* Test exact matching logic.
* Test fuzzy matching with edge cases (similar titles, author name variations).
* Test author-first constraint is enforced.

* [ ] **Task 5.3: Integration Tests**
* Test end-to-end flow with mock Calibre data.
* Verify tag format generation.

## 5. Data Schema Definition

### Canonical CSV Structure (`data/canonical_awards.csv`)

| Column | Type | Description | Example |
| --- | --- | --- | --- |
| `prize` | string | Name of the award | "Hugo Award" |
| `category` | string | Award sub-category | "Best Novel" |
| `year` | int | Year of award | 1966 |
| `status` | string | "Winner" or "Shortlist" | "Winner" |
| `title` | string | Canonical book title | "Dune" |
| `author` | string | Canonical author name | "Frank Herbert" |

### Calibre Tagging Taxonomy

The script will generate tags in this format:

`Award: Hugo [Winner]`, `Award: Booker [Shortlist]`, `Award: Giller [Nominee]` etc. (User configurable in `main.py`).

## 6. Incremental updates

If the script is interrupted, or if it is run again in the future, it should update the data rather than start over. Meaning, check if an entry for a previous year exists.

## 7. Error handling

Update separate logs for each service run, and also log to the console. Explode on bad errors, do not suppress them.

## 8. Scope & Constraints

* **Single Calibre library:** The script assumes one default Calibre library. No `--library-path` option.
* **No manual blocklist:** No mechanism to exclude false positive matches. If fuzzy matching produces errors, adjust thresholds.
* **Subtitle stripping is safe:** Edge cases where subtitle removal causes ambiguity are rare enough to ignore.
